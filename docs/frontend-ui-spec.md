# 前端页面描述规范与首批页面草案

> 日期：2026-03-27
> 用途：在终端环境中，用统一格式清晰描述页面布局、信息层级和交互
> 关联文档：`docs/architecture.md`、`docs/mcp-skill-integration.md`

## 1. 为什么要先定义描述规范

我们当前在终端里讨论前端，没有可视化画板，也不直接写代码。
所以前端设计文档必须满足两个条件：

- 读起来像页面，而不是抽象需求列表
- 足够结构化，后面能直接转成设计稿或前端实现

因此，后续所有页面都使用同一套文本格式描述。

## 2. 页面描述格式

每个页面都按以下结构书写：

### 2.1 页面卡片

- 页面名
- 目标用户
- 页面目标
- 页面入口
- 核心动作

### 2.2 信息结构

说明页面分成哪些区域，例如：

- 全局侧边栏
- 页面顶部栏
- 主内容区
- 右侧详情区
- 抽屉 / 弹窗

### 2.3 桌面端线框

使用等宽 ASCII 草图描述主布局。

### 2.4 关键交互

说明：

- 点哪里
- 展开什么
- 如何切换
- 哪些操作会打开抽屉或弹窗

### 2.5 状态设计

至少列出：

- 空状态
- 加载状态
- 成功状态
- 错误状态
- 权限不足状态

### 2.6 移动端规则

如果页面在移动端要折叠或改成抽屉，也要写明。

## 3. 全局信息架构草案

### 3.1 侧边栏导航结构

侧边栏建议采用“固定导航 + 分组列表 + 底部账户区”的结构。

导航项如下：

- `工作台`
- `新建对话`
- `探索技能`
- `资源库`
- `知识库`
- `项目`
- `最近`
- `定时任务`

说明：

- `工作台`
  - 作为首页
  - 包含一个显眼的对话框入口，可直接开启临时会话
  - 同时展示 overview 信息和常用导航
- `新建对话`
  - 本质上也是开启一个新的临时会话
  - 是一个高频主动作，不应埋在工作台内部
- `探索技能`
  - 即 `技能中心`
  - 进入后管理 `MCP`、`Skills`、绑定关系
- `资源库`
  - 存放对话、项目、工作流中生成的各种资产
  - 例如文档、摘要、图表、报告、导出结果
- `知识库`
  - 企业知识库入口
  - 偏向检索、浏览、授权后的知识访问
- `项目`
  - 正式 `Project` 列表
  - 组标题右侧应有 `新建项目` 按钮
- `最近`
  - 主要显示最近的对话历史
  - 侧重个人最近访问
- `定时任务`
  - 用于设置周期任务、定时执行和自动化调度

### 3.3 底部账户区

侧边栏最底部固定一个账户区，显示：

- 用户头像
- 用户名称
- 展开箭头

点击后弹出一个小窗口，至少包含：

- `设置`
- `帮助`
- `退出登录`

如有需要，也可以追加：

- 当前组织
- 当前环境
- 主题切换

### 3.5 侧边栏交互原则

- `工作台` 和 `新建对话` 必须都能进入临时会话，但语义不同：
  - `工作台` 是首页入口
  - `新建对话` 是直接动作
- `项目` 是一个带列表的分组，不只是单个页面入口
- `最近` 默认展示最近会话，不和 `项目` 历史混成一类
- 底部账户区必须固定，不随中间导航滚动
- 高优先级动作应尽量在侧边栏一跳可达

## 4. 页面一：技能中心

### 4.1 页面卡片

- 页面名：`技能中心`
- 目标用户：管理员、高级用户、项目负责人
- 页面目标：统一管理 `MCP`、`Skill` 以及它们和 `Agent` / `AgentTeam` / `Project` 的绑定关系
- 页面入口：全局侧边栏
- 核心动作：
  - 浏览已接入的 MCP
  - 浏览已安装的 Skills
  - 查看依赖关系
  - 建立绑定
  - 解除绑定
  - 查看最近调用状态

### 4.2 页面结构

页面建议采用三栏结构：

- 左侧：全局导航侧边栏
- 中间：列表区
- 右侧：详情与绑定区

页面顶部有二级标签：

- `MCP`
- `Skills`
- `Bindings`

### 4.6 关键交互

#### 交互 1：从 MCP 绑定到 Team

1. 用户进入 `技能中心`
2. 选择 `MCP` 标签
3. 点击某个 MCP，比如 `GitLab MCP`
4. 在右侧详情区点击 `绑定到 Team`
5. 打开绑定抽屉
6. 选择目标 Team，例如 `软件开发 Team`
7. 选择允许的命名空间或工具范围
8. 点击确认

#### 交互 2：从 Skill 绑定到 Agent

1. 用户进入 `Skills`
2. 选择某个 skill
3. 点击 `绑定到 Agent`
4. 弹出目标选择器
5. 选择某个 Agent
6. 保存

#### 交互 3：查看绑定是否生效

1. 用户进入 `Bindings`
2. 通过筛选器选择 `Team`
3. 查看 `软件开发 Team`
4. 展开看到：
   - 绑定了哪些 skill
   - 依赖了哪些 MCP
   - 最近一次调用是否成功

### 4.8 状态设计

#### 空状态

- 还没有接入任何 MCP
- 还没有安装任何 Skill
- 还没有建立任何绑定

文案应引导用户：

- 去接入第一个 MCP
- 去安装或启用 Skill
- 去完成第一条绑定

#### 加载状态

- 左侧列表使用骨架屏
- 右侧详情区使用占位块

#### 错误状态

- MCP 连接失败
- Skill 元数据加载失败
- 绑定保存失败

错误区应包含：

- 错误摘要
- 重试按钮
- 最近错误时间

#### 权限不足状态

如果当前用户没有管理技能中心权限：

- 页面仍可见，但为只读
- 绑定按钮置灰
- 展示提示：`你没有修改技能绑定的权限`

### 4.9 移动端规则

移动端不保留三栏。

改成两层结构：

- 顶部标签切换：`MCP / Skills / Bindings`
- 列表页进入详情页

绑定操作使用全屏抽屉。

### 4.10 设计结论

`技能中心` 不应该只是一个“插件列表页”，而应该是：

- 技能资产中心
- MCP 接入中心
- 绑定关系中心

这样用户才能真正理解：

- 系统接了什么能力
- 哪些 Agent / Team 正在使用这些能力
- 这些能力是否已在项目里生效

## 5. 后续页面建议顺序
