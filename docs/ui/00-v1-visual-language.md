# V1 视觉语言最小规范

> 日期：2026-04-01
> 用途：在前期资源有限的前提下，定义基于 shadcn/ui 的第一版视觉语言边界
> 适用范围：`SwarmMind/ui/` 第一版 Supervisor UI

## 1. 设计结论

SwarmMind V1 直接采用 `shadcn/ui` 作为组件底座，不追求首版就建立完整品牌视觉系统。

但即使是 V1，也必须先定义最小视觉语言，避免界面退化成没有状态语义的通用后台模板。

V1 原则：

- 组件层直接使用 `shadcn/ui`
- 视觉层只做最少量定制
- 优先统一 token、状态色和页面层级
- 暂不追求高度品牌化，但必须保留后续升级空间

## 2. 产品气质关键词

V1 统一气质如下：

- `Structured`
- `Operational`
- `Trustworthy`
- `High-signal`
- `Human-supervised AI`

产品感受应更接近“企业控制台”，而不是消费级聊天应用。

## 3. 组件策略

- 基础组件：使用 `shadcn/ui`
- 图标：使用 `lucide`
- 基础样式：沿用当前 `base-nova` preset
- 不在 V1 重写组件库，只在 token、布局和状态表达上做约束

工程要求：

- 所有颜色、圆角、阴影优先走 CSS variables，不写死在业务组件里
- 优先通过语义 token 扩展，而不是在页面里临时写一套新颜色

## 4. 字体与密度

- 主字体：`Geist`
- 等宽字体：`Geist Mono`
- 页面密度：中高密度
- 列表、看板、状态卡允许较高信息密度，但不得牺牲可扫描性

排版规则：

- 页面标题明确、克制，不做 oversized hero
- 项目页优先突出状态、风险、审批、产物，不追求留白感
- ChatSession 可以更轻，Project 页面必须更稳、更正式

## 5. 色彩与语义

V1 保持中性底色，不引入复杂品牌色系统。

基础方向：

- 背景：neutral / near-white
- 卡片：白底或极浅灰
- 文本：高对比深灰
- 边框：轻边框，弱阴影

必须先统一的状态语义：

- `running`：蓝色系
- `blocked`：橙色系
- `approval`：琥珀色系
- `risk-high`：红色系
- `done`：绿色系
- `draft` / `chat`：灰色系

要求：

- 状态色只用于状态，不拿来做大面积装饰
- 同一状态在工作台、项目页、审批中心必须保持一致

## 6. 页面层级规则

V1 至少区分两种页面气质：

### 6.1 ChatSession

- 更轻
- 更接近普通 AI chat
- 强调输入、摘要、提升为 Project
- 弱化治理感

### 6.2 Project

- 更正式
- 强调状态条、审批、风险、看板、产物
- 右侧区域优先承担上下文与治理信息
- 必须明显区别于普通聊天页面

一句话：

- `ChatSession` 是探索界面
- `Project` 是执行控制面

## 7. 卡片与面板语言

V1 统一使用 shadcn 风格卡片，但要遵守以下规则：

- 卡片是信息容器，不是装饰块
- 页面内主次层级最多三层
- 风险、审批、阻塞卡片优先使用状态标签和边框变化，而不是重背景色
- 项目状态条必须高于普通内容卡片

不做的事情：

- 不做复杂玻璃拟态
- 不做大面积渐变背景
- 不做花哨动画

## 8. V1 不做什么

以下内容明确延后：

- 完整品牌视觉系统
- 独特插画体系
- 高定制动效语言
- 复杂品牌色分层
- 组件级大规模重皮肤

## 9. 升级接口

虽然 V1 直接用 shadcn/ui，但要为 V1.5 保留升级空间：

- token 层先行
- 状态语义先行
- Chat 与 Project 的页面气质先分开
- Team、审批、风险、回放这些关键对象先有稳定视觉语法

这样后续即使做品牌升级，也不需要推翻整个组件实现。
