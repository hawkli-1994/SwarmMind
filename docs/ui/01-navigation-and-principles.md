# 全局导航与交互原则

> 日期：2026-04-03
> 用途：定义所有页面共享的外壳、导航、信息层级和跨页规则

> 视觉、字体、颜色、圆角、阴影、动效统一以 [`DESIGN.md`](../../DESIGN.md) 为准。

## 1. 设计定位

SwarmMind 的 UI 是三层控制面的组合，不是聊天工具外面套一点项目管理：

- 会话层：探索与发起
- 项目层：执行与协作
- 治理层：审批、技能、资产与权限聚合

全局必须体现这五条：

- `Project` 是唯一正式工作边界
- `Agent Team` 是项目入口，不是普通 Bot
- 审批是工作流中的显式暂停节点
- Artifact、Run、Audit 必须可见且可追溯
- 工作台是权限化驾驶舱，不另做第二套高层首页

```text
|------------------------------|

| - GitLab 接入方案            |
|------------------------------|
| 定时任务                     |
|------------------------------|
| [头像] Kr Li            v    |
+------------------------------+
```

### 2.2 导航原则

- `工作台` 是首页，不等于项目页
- `+ 新建对话` 是即时动作，进入轻量 `ChatSession`
- `项目` 是分组，不是单一入口
- `最近` 只承接个人最近访问，不和项目列表混排
- 账户区固定在底部，不随导航中部滚动

## 3. 全局页面骨架

```text
+------------------------------------------------------------------------------------------------------------------+
| 侧边栏                                                                 | 顶栏：面包屑 / 搜索 / 主动作              |
+--------------------------------+---------------------------------------------------------------------------------+
| 导航与项目分组                 | 页面主体                                                                         |
|                                |----------------------------------------------------------------------------------|
|                                | 左：列表 / 辅助导航                                                               |
|                                | 中：主工作区                                                                     |
|                                | 右：详情 / 上下文 / 审批 / 运行状态                                              |
|                                | 底：状态栏 / 快捷输入 / 流式日志                                                 |
+--------------------------------+---------------------------------------------------------------------------------+
```

规则：

- 不是每页都必须三栏，但尽量共享同一栅格逻辑
- 右栏默认承担详情、治理、上下文
- 主动作始终在顶栏右上角

## 4. 关键全局组件

### 4.1 账户区

账户区至少承接：

- 设置
- 帮助
- 切换组织
- 退出登录

### 4.2 全局命令面板

建议统一提供 `Cmd/Ctrl + K`，覆盖：

- 搜索项目
- 搜索最近会话
- 跳转审批
- 新建项目
- 新建对话
- 跳转技能中心

### 4.3 全局状态信息

右上角保留轻量系统状态：

- 连接状态
- 当前组织
- 待审批数量
- 正在运行项目数

## 5. 跨页交互原则

### 5.1 ChatSession 到 Project

- 轻量聊天是一等入口，不是创建项目之前的占位页
- 用户可直接用它完成提问、头脑风暴、快速分析和临时任务
- 一旦涉及共享、审批、Team、正式工作流，应引导 `提升为 Project`
- 提升动作写入的是语义摘要，不是机械复制整段聊天

口径：

- `Chat first for exploration`
- `Project first for execution and governance`

### 5.2 Project 内统一入口

- 项目内聊天默认连接当前 `Agent Team` 的项目实例
- 用户进入的是“项目协作台”，不是孤立对话窗口
- 入口消息要回写看板、时间线、风险和 artifact，而不是只追加聊天记录

### 5.3 审批中断

- 审批触发后，项目必须显式进入暂停状态
- 审批结果必须回流项目并支持恢复或阻塞说明

### 5.4 技能生效闭环

- 能力配置在 `技能中心`
- 生效反馈必须在 `项目` 与 `审批` 页面里可见
- 不允许用户跨多个设置页猜测绑定是否成功

## 6. 工作台与项目空间约束

### 6.1 工作台

- 不单独拆高层驾驶舱页面
- 差异来自权限和角色，不来自另一套页面结构
- 工作台卡片只分三类：
  - 固定卡片
  - 权限卡片
  - 可配置卡片
- 无权限或无数据时优先隐藏或降级，不保留空壳

### 6.2 项目空间

- 所有正式 `Project` 页面共享固定骨架
- 不允许每个子页重新发明项目导航
- 固定骨架至少包含：
  - 项目顶栏
  - 项目状态条
  - 二级导航
  - 子页内容区
- 权限影响可见深度，不影响导航顺序

## 7. 范围约束

- 不新增新的顶层导航入口
- 不新增新的项目二级 tab
- 缺失能力优先落在：
  - 现有页面卡片
  - 抽屉 / 侧栏
  - 过滤器
  - 完成态 / 空状态
- 只有当现有承载方式无法胜任时，才新增独立页面

## 8. 响应式总规则

- 平板：侧边栏可折叠，右栏优先改抽屉
- 手机：主内容单列，详情转全屏抽屉或二级页
- 移动端优先保留：列表、状态、主动作
- 移动端优先弱化：复杂三栏、长表格、多图表并列
