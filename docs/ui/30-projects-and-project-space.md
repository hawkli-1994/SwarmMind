# 项目体系与项目空间

> 日期：2026-04-03
> 用途：定义从项目创建到项目执行、看板、产物和设置的整套页面

## 0. 项目空间固定结构

项目空间的固定导航已在 `docs/ui/02-page-map-and-flows.md` 定义。
本页只保留项目域自己的约束和页面细化。

项目空间不是离散 tab 集合，而是一个共享骨架下的正式工作空间。

固定骨架：

```text
+------------------------------------------------------------------------------------------------+
| 项目顶栏：项目名 / 状态 / 当前阶段 / 主动作                                                    |
| 项目状态条：阻塞 / 待审批 / 风险 / 活跃 Team                                                   |
| 二级导航：[总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                  |
+------------------------------------------------------------------------------------------------+
| 当前子页内容                                                                                    |
+------------------------------------------------------------------------------------------------+
```

固定约束：

- 二级导航顺序固定。
- 权限影响内容深度，不改变导航结构。
- 新能力优先归入现有六个子页，不轻易新增平行页面。
- 跨项目治理动作，例如统一审批，优先放在项目外的治理入口中。
- 项目进入 `已完成` 后，仍沿用同一骨架，但主动作和状态文案切到归档与回放语义。
- 项目相关的“回放”“审批跟踪”“Team 挂载”优先做成现有子页中的抽屉或状态区，不新增新 tab。

六个子页各自回答的问题：

| 子页 | 用户核心问题 | 默认对象 |
|------|--------------|----------|
| 总览 | 这个项目现在整体什么状态？ | 所有人 |
| 协作台 | 我要如何继续推动它？ | 项目成员 |
| 看板 | 任务具体流到哪一步了？ | 项目成员 / 负责人 |
| 时间线与风险 | 节奏、依赖和风险在哪里？ | 负责人 / 管理者 |
| 产物 | 已经沉淀了什么正式输出、审计和回放？ | 有访问权限的成员 |
| 设置 | 这个项目允许谁参与、用什么能力？ | 负责人 / 管理员 |

项目顶栏草图：

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设      进行中 / 需求澄清      [进入协作台] [上传资料] [更多操作]             |
| 目标：构建企业统一 CRM，首期交付销售 CRM MVP                                                  |
| 阻塞：1   待审批：1   风险：medium   活跃 Team：软件开发 Team                                  |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------------------------------------------------------------------------+
```

导航和内容规则：

- `总览` 永远是默认落点
- 顶栏主动作优先导向 `进入协作台`
- 阻塞、待审批、风险必须在所有子页可见
- 右侧位按子页复用摘要、治理、详情和回放入口
- 项目完成后，默认仍落 `总览`，但首屏切成“完成摘要 + 归档 + 回放”

## 1. 页面一：项目列表

### 1.1 页面卡片

- 页面名：`项目列表`
- 目标用户：所有有项目访问权限的成员
- 页面目标：浏览、筛选、进入正式项目空间
- 页面入口：侧边栏 `项目`
- 核心动作：
  - 查看项目状态
  - 新建项目
  - 按 Team、状态、风险筛选

### 1.2 信息结构

- 顶部：搜索与筛选
- 中部：项目表格 / 卡片
- 右侧：项目摘要详情

### 1.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 项目                                                           [搜索项目] [筛选] [新建项目]    |
+------------------------------------------------------------------------------------------------+
| 名称                    | Team              | 当前阶段     | 状态       | 风险     | 操作     |
| 新一代 CRM 平台建设     | 软件开发 Team     | 需求澄清     | 进行中     | medium   | 打开     |
| 招聘流程优化            | 招聘 Team         | 面试安排     | 等待审批   | high     | 打开     |
| 销售培训体系            | 销售培训 Team     | 验收         | 已完成     | low      | 打开     |
+------------------------------------------------------------------------------------------------+
| 右侧摘要：项目目标 / 负责人 / 最近 artifact / 待审批数量 / 当前阻塞                            |
+------------------------------------------------------------------------------------------------+
```

### 1.4 关键交互

1. 点击项目行进入项目总览。
2. 点击 `新建项目` 进入创建页。
3. 筛选项支持 `状态 / Team / 成员 / 风险 / 最近更新`。

补充说明：

- `成员` 筛选用于承接工作台中的 `项目成员参与分布` 卡片下钻。
- 点击某位成员后，项目列表自动过滤出该成员参与的项目，并在右侧摘要区显示该成员在各项目中的角色。

### 1.5 状态设计

- 空状态：引导创建第一个项目
- 加载状态：表格骨架
- 成功状态：支持批量浏览和快速筛选
- 错误状态：项目列表与右侧摘要分别可重试
- 权限不足状态：隐藏无权限项目，只保留可访问项目

### 1.6 平台范围约束

- 当前版本只定义桌面端项目列表，不提供移动端独立方案
- 视口缩窄时优先压缩筛选区与摘要区，不改变“列表 + 详情”的主结构

## 2. 页面二：新建项目

### 2.1 页面卡片

- 页面名：`新建项目`
- 目标用户：项目发起人、管理员
- 页面目标：创建正式 `Project`，并允许立即挂载或稍后挂载 Team 模板 / Workflow
- 页面入口：工作台、项目列表、ChatSession 提升
- 核心动作：
  - 填写目标 / 范围 / 约束
  - 设置可见范围
  - 选择 Agent Team 模板
  - 创建项目

### 2.2 信息结构

- 左侧：项目表单
- 右侧：创建预览与默认 Team 说明

### 2.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 新建项目                                                                            [创建项目] |
+-------------------------------------------------------------+----------------------------------+
| 基本信息                                                    | 创建预览                         |
| 项目名称 [ 新一代 CRM 平台建设                        ]     | Team：软件开发 Team              |
| 项目目标 [ 构建企业统一 CRM                        ]        | 默认流程：软件开发默认工作流     |
| 业务范围 [ 客户/联系人/商机/报表...               ]        | 将生成：Overview / Kanban / Artifact |
| 约束     [ Web 优先 / 3 个月 MVP / 接入 SSO        ]        | 可见范围：业务/技术/管理层        |
| 可见范围 [ 业务部 ][ 技术部 ][ 管理层 ]                    | 来源：来自 ChatSession 摘要       |
| 选择 Team 模板 [ 软件开发 Team                    v ]       |                                  |
| [稍后在项目设置中挂载 Team 模板]                             | [预览工作流]                     |
+-------------------------------------------------------------+----------------------------------+
```

### 2.4 关键交互

1. 从 ChatSession 提升进入时，目标、范围、约束自动预填。
2. 选择 Team 模板后，右侧同步显示默认 workflow 说明。
3. 点击 `创建项目` 后直接进入项目总览。
4. 若点击 `稍后在项目设置中挂载 Team 模板`，则先创建空项目，项目总览显示“尚未挂载 Team”的启动态。

### 2.5 状态设计

- 空状态：展示内置 Team 的典型用途
- 加载状态：Team 列表和默认 workflow 预览加载中
- 成功状态分两类：
  - 已选择 Team：展示“项目已建立，项目内 Team 实例已上线”
  - 稍后挂载 Team：展示“项目已建立，尚未挂载 Team 模板，请前往设置完成挂载”
- 错误状态：字段校验错误、Team 加载错误、创建失败
- 权限不足状态：没有项目创建权限时只展示申请入口

说明：

- `Team 模板` 在创建页推荐选择，但不是强制前置条件。
- 这样可以兼容用户故事里“先建 Project，再去设置页挂 Team 模板”的路径。

### 2.6 平台范围约束

- 当前版本只定义桌面端项目创建页，不提供移动端独立方案
- 视口缩窄时优先压缩辅助说明与预览区，不改单列桌面表单逻辑
- `创建项目` 固定底部

## 3. 页面三：项目总览

### 3.1 页面卡片

- 页面名：`项目总览`
- 目标用户：项目成员、审批人、管理者
- 页面目标：用一个页面概览项目状态、阶段、阻塞、产物和成员
- 页面入口：项目列表、工作台
- 核心动作：
  - 查看当前阶段
  - 打开项目协作台
  - 进入看板、时间线、产物
  - 发现阻塞与审批

### 3.2 信息结构

- 顶部：项目顶栏、状态条、固定二级导航
- 中部：阶段摘要卡、阻塞卡、待审批卡、成员卡
- 下部：最近任务、最近 artifact、运行时间线摘要、审批与决策摘要
- 浮层：项目回放抽屉

### 3.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设                      进行中 / 需求澄清          [进入协作台] [更多操作]    |
| 阻塞：1   待审批：1   风险：medium   活跃 Team：软件开发 Team                                  |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------------------------------------------------------------------------+
| 目标：构建企业统一 CRM，首期交付销售 CRM MVP                                                  |
|------------------------------------------------------------------------------------------------|
| 阶段进度              | 当前阻塞                 | 待审批                | 参与成员              |
| 需求澄清 -> 设计 -> 开发 | 知识库访问未获批准        | 1 个访问审批待处理     | 3 人类 + 7 个 Agent   |
|------------------------------------------------------------------------------------------------|
| 最近任务                                      | 最近产物                                  |
| - 输出 PRD 草案                               | - PRD v0.3                               |
| - 设计 CRM 模块边界                           | - 架构草图                               |
| - 识别 SSO 接入方案                           | - 风险清单                               |
|------------------------------------------------------------------------------------------------|
| 审批与决策摘要                                                                                 |
| - Internal KB 访问审批：已提交给 企业高层 A                                                     |
| - MVP 报表范围已缩减为销售漏斗汇总                                                             |
| [查看审批跟踪] [查看完整回放]                                                                  |
|------------------------------------------------------------------------------------------------|

+------------------------------------------------------------------------------------------------+
```

### 3.4 关键交互

1. 点击 `进入协作台`，进入当前 `Agent Team` 的项目协作入口。
2. 点击阻塞卡或待审批卡：
   - 审批人进入可操作审批详情
   - 请求发起人进入只读审批跟踪抽屉
3. 点击最近 artifact，进入项目产物库对应条目。
4. 点击 `查看完整回放`，打开项目级 run / audit 时间线。

### 3.5 状态设计

- 空状态分两类：
  - 尚未挂载 Team：提示“先去项目设置挂载 Team 模板，再创建项目内团队实例”
  - 已挂载 Team 但尚未启动：提示“向当前 Agent Team 说明需求”
- 加载状态：各摘要卡独立骨架
- 成功状态：展示阶段、任务、artifact、成员健康度
- 错误状态：局部模块失败不阻断整个总览
- 权限不足状态：隐藏受限 artifact 和成员信息

未挂载 Team 的启动态草图：

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设                      未启动 / 尚未挂载 Team          [前往项目设置]         |
| 阻塞：0   待审批：0   风险：none   活跃 Team：无                                               |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------------------------------------------------------------------------+
| 当前项目还没有挂载 Team，因此还没有项目内 Team 实例、工作流和任务骨架。                        |
| 下一步：进入设置页挂载一个可复用 Team，例如 `软件开发 Team`。                                  |
| [挂载第一个 Team]                                                                              |
+------------------------------------------------------------------------------------------------+
```

### 3.6 完成态

项目完成后，总览应切换为完成态：

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设                      已完成 / MVP 交付完成        [查看项目回放] [更多操作] |
| 审批链路：完整   风险：已收敛   关键产物：已归档   Team 经验：待沉淀 / 已沉淀                    |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------------------------------------------------------------------------+
| 完成摘要：MVP 已交付，核心模块、SSO、基础报表已完成                                             |
| 关键成果：PRD / 架构图 / 前端 IA / 接口说明 / 测试报告                                         |
| [查看归档产物] [沉淀为 Team 资产]                                                                |
+------------------------------------------------------------------------------------------------+
```

说明：

- 看板显示全部收敛。
- 时间线显示主要阶段完成。
- 协作台转只读，不再鼓励继续追加执行型消息。

### 3.7 项目回放抽屉

为控制范围，`项目回放` 不做独立页面，先作为 `总览` 与 `产物` 共用的抽屉。

抽屉内容最小包括：

- 关键阶段时间线
- 重要 `Run`
- 关键 `Artifact`
- 审批节点
- 决策摘要
- `actor_id`
- `interface_agent`
- `target_agent`
- 关键 handoff

```text
+------------------------------------------------------------------+
| 项目回放                                                      X |
|------------------------------------------------------------------|
| 项目启动 -> 需求澄清 -> 架构设计 -> 开发执行 -> 测试验收 -> 完成 |
|                                                                  |
| 关键 Run                                                         |
| - run-01 需求 intake                                              |
|   actor：业务员工用户                                             |
|   project_team_instance：软件开发 Team / CRM MVP                  |
|   target_agent：产品经理 Agent                                    |
| - run-08 Internal KB 审批触发                                      |
|   actor：架构师 Agent                                              |
|   decision：escalate                                              |
|   target_agent：审批流                                             |
| - run-15 MVP 交付总结                                              |
|   actor：交付协调 Agent                                            |
|   target_agent：项目归档                                            |
|                                                                  |
| 关键审批                                                         |
| - Internal KB 访问：已批准                                        |
|   approver：企业高层 A                                             |
|                                                                  |
| 关键 Handoff                                                     |
| - 产品经理 Agent -> 架构师 Agent：范围切分完成                    |
| - 架构师 Agent -> Tech Lead Agent：模块边界确认                   |
| - Tech Lead Agent -> 前后端 Agent：进入开发执行                    |
|                                                                  |
| 关键产物                                                         |
| - PRD v0.3                                                        |
| - 架构图                                                          |
| - 测试报告                                                        |
+------------------------------------------------------------------+
```

交互原则：

- 从总览点击 `查看完整回放` 打开
- 从产物页点击 `查看项目回放` 打开
- 已完成项目默认优先展示最终摘要，而不是原始逐条日志
- 默认展示摘要级回放，但用户应可展开查看 `actor / decision / agent / handoff` 细节

### 3.8 平台范围约束

- 当前版本只定义桌面端项目总览，不提供移动端独立方案
- 视口缩窄时优先收缩次级摘要区，不改变总览卡片与时间线的桌面信息组织

## 4. 页面四：项目协作台

### 4.1 页面卡片

- 页面名：`项目协作台`
- 目标用户：项目成员
- 页面目标：作为当前 `Agent Team` 的统一协作入口，承接需求、干预、补充和派发；项目完成后转为只读回看
- 页面入口：项目总览 `进入协作台`
- 核心动作：
  - 向当前 Agent Team 提交指令
  - 查看工作流归属与运行决策
  - 追踪当前活跃 Agent
  - 触发新任务或调整范围

### 4.2 信息结构

- 顶部：项目顶栏、状态条、固定二级导航
- 左侧：Workstream 列表
- 中间：项目对话 / 活动流
- 右侧：上下文摘要、当前运行上下文、路由决策

### 4.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设 / 协作台                                  [上传资料] [查看运行日志]       |
| 阻塞：1   待审批：1   风险：medium   活跃 Team：软件开发 Team                                  |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------+--------------------------------+--------------------------------+
| Workstreams                  | 协作流                         | 上下文与治理                     |
| - 项目启动                    | 用户：MVP 先别做复杂报表      | 当前 Workstream：项目启动        |
| - 权限与知识库接入            | Agent Team：已调整范围        | target_role：产品专家            |
| - 前端信息架构                | 并生成新任务“基础销售漏斗汇总”| decision：reuse_thread           |
|                                |                                | runtime context：ws-init-003     |
|                                |                                | 可见级别：project-internal       |
|                                | [输入框......................................................] |
|                                |                                | [新建工作流] [查看线程详情]      |
+------------------------------+--------------------------------+--------------------------------+
```

### 4.4 关键交互

1. 用户在输入框提交指令，右栏立即显示 `InterfaceDecision` 摘要。
2. 如果命中新工作流，左侧自动新增 Workstream。
3. 若触发高风险能力，顶部插入“等待审批”条。
4. 审批中心中选择 `要求补充` 后，该请求会以“待补充问题”形式回流到当前 Workstream。

### 4.5 状态设计

- 空状态：只显示项目启动提示和示例输入
- 加载状态：活动流流式刷新，右栏显示路由中
- 成功状态：可以看到指令如何变成任务与工作流
- 错误状态：路由失败、thread 冲突、权限不足分别提示
- 权限不足状态：没有 `project.run` 时输入框禁用

补充约束：

- `thread` 属于内部运行锚点，只能在调试、审计或治理上下文中以辅助信息出现。
- 项目成员主视图默认看到的是“运行上下文”或“协作决策”，而不是 DeerFlow 底层术语。

完成态补充：

- 项目完成后，输入框隐藏
- 页面切换为只读协作流
- 主操作改为 `查看项目回放`

### 4.6 平台范围约束

- 当前版本只定义桌面端项目协作台，不提供移动端独立方案
- 视口缩窄时优先折叠次级栏位，不把 Workstream 和上下文改写成移动端抽屉流

## 5. 页面五：项目看板

### 5.1 页面卡片

- 页面名：`项目看板`
- 目标用户：项目成员、管理者

3. 支持按任务、作者、时间和类型筛选。
4. 已完成项目中，支持从产物库直接打开“项目完整回放”。

### 7.5 状态设计

- 空状态：提示“项目推进后，产物会自动归档到这里”
- 加载状态：列表与预览分别骨架
- 成功状态：可追溯到任务、run、审批
- 错误状态：预览失败时保留下载入口
- 权限不足状态：只显示元数据，不开放内容预览

### 7.6 Team 资产沉淀

项目产物与 Team 资产必须分层。

在已完成项目中，产物库应支持：

- 选择某个 artifact 或复盘结论
- 点击 `沉淀为 Team 资产`
- 选择目标 Team
- 选择沉淀类型：`playbook / knowledge pack / 复盘模板`

沉淀完成后：

- 项目内保留原始 artifact
- Team 侧生成新的可复用资产引用
- 两者通过来源链接相互可追溯

范围约束：

- 这里不新增单独的 `TeamAssetCenter`
- 先把 Team 资产统一落到 `资源库` 的 `Team 资产` 视图中
- 后续如确实复杂，再考虑拆分

### 7.7 平台范围约束

- 当前版本只定义桌面端产物与资源视图，不提供移动端独立方案
- 视口缩窄时优先压缩元数据区和预览宽度，不改写查看器的桌面主结构
