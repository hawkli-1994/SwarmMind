# 页面地图与关键流程

> 日期：2026-04-03
> 用途：定义页面全集与主流程，避免页面文档变成孤立线框

## 1. 顶层页面地图

```text
工作台
  |- 新建对话
  |- 最近记录
  |- 待审批入口
  |- 项目快捷入口

探索技能
  |- MCP
  |- Skills
  |- Bindings

项目
```

## 2. 关键流转总览

SwarmMind 目前只维护这六条主流程：

1. 探索聊天 -> 正式项目
2. 项目启动 -> 持续推进
3. 审批暂停 -> 审批恢复
4. 技能治理 -> 项目生效
5. 项目完成 -> 资产沉淀
6. 权限化工作台巡视 -> 项目下钻

新增页面或功能时，优先挂到这六条主流程里，不再扩张新的平行主线。

## 3. 主流程一：探索聊天到正式项目

```text
工作台
  -> 新建对话
  -> ChatSession 形成问题摘要
  -> 用户点击 提升为 Project
  -> 新建项目页预填目标 / 范围 / 约束
  -> 选择 Agent Team 模板
  -> 进入项目总览
  -> 项目协作台开始正式 intake
```

规则：

- 提升动作必须显眼，但不强迫
- 项目创建成功后，原 ChatSession 变为只读来源记录

## 4. 主流程二：项目启动到持续推进

```text
项目列表
  -> 新建项目
  -> 选择 Agent Team 模板
  -> 创建 ProjectAgentTeamInstance
  -> 自动挂载默认 WorkflowTemplate
  -> 项目空间 / 总览
  -> 项目协作台
  -> 自动生成任务骨架 / 看板 / 初始 artifact
  -> 项目看板 / 时间线 / 产物库持续更新
```

规则：

- 创建阶段不重复要求用户选择几乎同义的对象
- 协作台 intake 结果必须回写总览、看板、时间线和产物
- 创建时未选 Team，项目设置页必须允许后挂并完成实例生成

### 4.1 项目空间固定导航

```text
项目空间
  |- 总览
  |- 协作台
  |- 看板
  |- 时间线与风险
  |- 产物
  |- 设置
```

规则：

- 这六项构成固定二级导航
- 新能力优先挂在现有子页里，不新增平行 tab
- 审批通过状态卡、抽屉或侧栏回看，不变成常驻项目 tab
- 回放优先落在 `总览` 或 `产物` 的侧栏 / 抽屉里

## 5. 主流程三：审批暂停与恢复

```text
项目内操作触发高风险能力
  -> 项目状态变为 等待审批
  -> 审批中心收到审批项
  -> 审批人打开审批详情
  -> 批准 / 拒绝 / 要求补充
  -> AuditLog 写入结果
  -> 项目恢复推进或进入阻塞解释态
```

规则：

- 审批对象必须是具体请求，不是模糊聊天消息
- 拒绝后保留替代建议和影响说明
- 发起人必须能在项目里看到只读审批跟踪态

## 6. 主流程四：能力治理到项目生效

```text
技能中心
  -> 配置 MCP
  -> 安装 / 绑定 Skill
  -> 绑定到 Team 模板 / 项目 Team 实例 / Project
  -> 项目设置页可见已启用能力
  -> 项目协作台实际调用
  -> 审批中心 / 运行日志展示调用结果
```

规则：

- “配置能力” 与 “在项目里生效” 必须闭环

## 7. 主流程五：项目完成、归档与沉淀

```text
项目看板全部收敛
  -> 项目状态变为 已完成
  -> 项目总览显示完成摘要
  -> 项目产物归档到项目产物库
  -> 审批链路和运行回放可完整查看
  -> 用户将复盘结论沉淀为 Team 资产
  -> Team 资产进入可复用知识 / playbook
```

规则：

- 项目完成后必须有清晰完成态，不能只剩文件
- 项目产物与 Team 资产必须分层沉淀
- `Team` 资产优先进入 `资源库` 的筛选视图，不新增独立顶层入口

## 8. 主流程六：权限化工作台巡视与下钻

```text
工作台
  -> 按权限看项目组合状态
  -> 查看阻塞项目 / 待审批 / 风险热区
  -> 点入单个项目
  -> 查看项目总览 / 看板 / 审批 / artifact
  -> 回到工作台继续横向比较
```

规则：

- 工作台承担驾驶舱角色，但不替代项目页
- 高权限用户可快速下钻，但不应被迫卷入执行细节
- 管理视角优先看项目组合状态、审批堆积、风险与 Team 负载
